# 前端展示规则

## 基础展示模块

前端页面至少展示以下内容：

- 物流单号
- 承运商
- 当前状态
- 最近更新时间
- 起运地 / 目的地
- 物流轨迹时间线
- 支持提示文案

## 状态展示规则

- `info_received`
  使用偏中性或蓝绿色标签。
  提示文案：承运商尚未完成首次扫描。

- `in_transit`
  作为主流程状态展示。
  提示文案：包裹正在运输网络中流转。

- `out_for_delivery`
  使用偏暖色标签。
  提示文案：包裹已进入末端派送阶段。

- `delivered`
  使用成功态标签，并展示签收完成提示。
  提示文案：包裹已完成签收。

- `exception`
  使用高优先级异常标签，并把提示区域放在时间线前面。
  提示文案：物流异常需要关注。

- `failed_attempt`
  使用异常或警告标签。
  提示文案：本次投递未成功。

- `not_found`
  使用弱化标签。
  提示文案：暂时没有可展示的轨迹信息。

- `expired`
  使用弱化标签。
  提示文案：该单号已停止活跃更新。

- `unknown`
  使用兜底标签。
  必要时展示原始状态文本辅助判断。

## 交互规则

- 前端只依赖后端标准化后的字段，不直接使用 17TRACK 原始状态枚举。
- 前端可以展示 `provider_status` 和 `provider_status_description` 作为排查辅助信息，但不能用它们决定 UI 状态。
- 如果 `events` 为空，也要保留卡片主体，并展示支持提示文案。
- 如果一批查询中部分单号失败，成功结果照常展示，同时在页面顶部给出错误提示。

## 进度展示规则

基础进度阶段：

```text
已收到信息 -> 运输中 -> 派送中 -> 已签收
```

对应关系：

- `info_received`
  点亮第一段。

- `in_transit`
  点亮前两段。

- `out_for_delivery`
  点亮前三段。

- `delivered`
  点亮全部阶段。

- `exception`、`failed_attempt`、`not_found`、`expired`、`unknown`
  不强行套用进度条，优先展示状态标签和支持提示。
