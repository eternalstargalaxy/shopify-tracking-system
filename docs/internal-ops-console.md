# 内部物流查询台

## 目标

当前客服和运营查物流时，需要在 4PX、云途、17TRACK 等渠道之间来回切换。这个内部查询台的第一版，先复用已经跑通的物流查询后端，把查过的数据沉淀在本地缓存里，提供统一入口和批量查询能力。

## 当前能力

- 页面入口：`/internal`
- 批量查询：支持一行一个、空格分隔或整段文本自动提取物流单号
- 接口入口：`POST /internal/api/track`
- 最近查询：`GET /internal/api/recent`
- 查询结果缓存：复用 `tracking_records` 表
- CSV 导出：在浏览器端导出本次查询结果

## 安全方式

内部查询台不走 Shopify App Proxy，而是通过环境变量 `INTERNAL_DASHBOARD_TOKEN` 做轻量令牌校验。

- 页面本身可打开
- 查询和最近记录接口必须携带 `x-internal-token`
- 内部工具默认绕过“仅本店订单可查”限制，避免后续外部安全收口后影响客服使用

## 当前适用场景

- 客服快速查单
- 运营批量检查多个物流单号
- 用缓存加速重复查询
- 导出当前查询结果给同事继续处理
